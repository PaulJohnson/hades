Known Issues

Opening PDFs with file:// URLs in Adobe Acrobat

If you have Adobe Acrobat installed and use a file:// URL to refer to a PDF document then clicking on the link button in a matrix will cause Acrobat to report an error with the file name instead of opening it. At present the only workarounds we can offer are to uninstall Acrobat, configure another program as the default for opening PDFs, or not use file:// URLs with PDFs (other types of URL, such as http://, work fine).

This appears to be a conflict between Acrobat and the GLib open source library. We will investigate further and submit a bug report if appropriate.

Other Issues

If you encounter any other problems with using the DSM then please let us know. Email <support@diametricsoftware.com> with:

  • The version of the DSM you are using (look at the window title bar).
  • The steps to reproduce the problem.
  • The difference between what you expect and what actually happens.
  • If the problem is associated with a particular file then it would be very helpful if you could provide a copy. If this requires an NDA then we will be happy to consider it.
